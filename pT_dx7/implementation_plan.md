# Implementation Plan - DX7 Algorithm Graphical Representation

## 1. Architecture & Patterns

- **Style**: C, character-cell rendering via `chargfx` API. Matches existing codebase conventions (`snake_case`, `static` helpers, `CHARGFX_*` colors).
- **Key Files**:
  - `display/gen_algo.py` — NEW: build-time Python script
  - `display/algo_conn_table.h` — GENERATED: algorithm connectivity data
  - `display/algo_gfx.h` — NEW: algorithm diagram renderer declarations
  - `display/algo_gfx.c` — NEW: algorithm diagram renderer implementation
  - `display/CMakeLists.txt` — MODIFIED: add new source files
  - `display/ui_widgets.h` — MODIFIED: add `ui_widgets_draw_algo()` declaration
  - `display/ui_widgets.c` — MODIFIED: add `ui_widgets_draw_algo()` implementation
  - `display/display_startup.c` — MODIFIED: integrate algorithm diagram into `display_status_update()`
- **Data Models**:
  - `algo_connections[32][MAX_CONNS][2]` — uint8_t array, each entry is `{source_op, dest_op}`
  - Operator box layout: fixed positions in char-cell coordinates

## 2. Critical Corrections to Original Plan

The original `algo-plan2.md` had several issues I'm fixing:

1. **Python parsing logic is fundamentally wrong**: The original script tried to extract connections by finding all `SEL > 0` operators and connecting them all to the last `C == 1` operator. This produces incorrect results (e.g., for Algorithm 1 it would connect Op6→Op1, Op5→Op1, Op4→Op1, Op1→Op1 instead of the correct Op6→Op1, Op5→Op2, Op4→Op2). **Fix**: Hardcode the known algorithm connections directly in the Python script since there are only 32 algorithms.

2. **Algorithm 10-15 connections**: The original plan's example connections for algorithms 10-15 included incorrect "Op5→Op5" self-loops. **Fix**: These are simply algorithms where Op6 feeds Op5, Op4 feeds Op3, and Op2 feeds Op1.

3. **Drawing approach**: The original plan suggested using Unicode box-drawing characters. The available `FONT_SPECIAL_CHARACTERS_BITMAP` has different glyphs. **Fix**: Use the available special characters (indices 0-127 in the bitmap) for box drawing and line drawing.

### Phase 2: Core Rendering

- [ ] **Step 4**: Create `display/algo_gfx.h` — algorithm diagram renderer declarations
  - *Context*: Provides the public API for drawing the algorithm diagram.
  - *Instruction*: Create the header file with:
    ```c
    #ifndef DX_2350_ALGO_GFX_H
    #define DX_2350_ALGO_GFX_H

    #include <stdint.h>
    #include "chargfx.h"

    /**
     * Draw the DX7-style algorithm connectivity diagram.
     *
     * @param panel_x     top-left character column
     * @param panel_y     top-left character row
     * @param panel_w     panel width in characters (recommended: 30)
     * @param panel_h     panel height in characters (recommended: 9)
     * @param alg_idx     algorithm index 0..31
     * @param op_levels   pointer to 6 op_levels (0..99) for level bar backgrounds, or NULL
     * @param op_highlight 6-bit mask (bit N set = operator N+1 active)
     */
    void algo_draw(uint8_t panel_x, uint8_t panel_y, uint8_t panel_w, uint8_t panel_h,
                   uint8_t alg_idx,
                   const uint8_t op_levels[6],
                   uint8_t op_highlight);

    #endif /* DX_2350_ALGO_GFX_H */
    ```

- [ ] **Step 5**: Create `display/algo_gfx.c` — algorithm diagram renderer implementation
  - *Context*: This is the main rendering logic. Uses chargfx character-cell rendering with available special characters from `FONT_SPECIAL_CHARACTERS_BITMAP`.
  - *Instruction*: Create the C file with the following structure:
  
    **5a. Fixed operator box positions** (char-cell coordinates):
    ```c
    // DX7 standard layout (x, y, w, h, label_char)
    // Row 1 (top):    Op6  |  Op5
    // Row 2 (middle): Op4  |  Op3
    // Row 3 (bottom): Op2  |  Op1
    static const struct { uint8_t x, y, w, h; char label; } DX7_OPS[6] = {
        {15, 8,  6, 3, '6'},  // Op6  — top-left
        {23, 8,  6, 3, '5'},  // Op5  — top-right
        { 7, 15, 6, 3, '4'},  // Op4  — mid-left
        {15, 15, 6, 3, '3'},  // Op3  — mid-right
        { 7, 22, 6, 3, '2'},  // Op2  — bot-left
        {15, 22, 6, 3, '1'},  // Op1  — bot-right
    };
    ```
    
    **5b. Special character glyphs** (mapped to FONT_SPECIAL_CHARACTERS_BITMAP indices):
    ```c
    // Glyph constants for box drawing and arrows
    // These map to indices in FONT_SPECIAL_CHARACTERS_BITMAP
    #define GLYPH_BOX_FULL       (char)0x80  // index 0: full box outline
    #define GLYPH_BOX_FILLED     (char)0x81  // index 1: filled box
    #define GLYPH_BOX_VERT       (char)0x82  // index 2: vertical box with sides
    #define GLYPH_BOX_HORIZ      (char)0x83  // index 3: top and bottom lines
    #define GLYPH_CORNER_BL      (char)0x84  // index 4: bottom-left corner
    #define GLYPH_CORNER_BR      (char)0x85  // index 5: bottom-right corner
    #define GLYPH_CORNER_TL      (char)0x86  // index 6: top-left corner
    #define GLYPH_CORNER_TR      (char)0x87  // index 7: top-right corner
    #define GLYPH_ARROW_RIGHT    '>'  // ASCII
    #define GLYPH_ARROW_DOWN     'v' // ASCII
    #define GLYPH_ARROW_DIAG     '/' // ASCII diagonal
    #define GLYPH_LINE_HORIZ     '-' // ASCII horizontal
    #define GLYPH_LINE_VERT      '|' // ASCII vertical
    ```
    *Note*: The actual glyph indices in `FONT_SPECIAL_CHARACTERS_BITMAP` need to be verified against the bitmap data. The coder should examine the bitmap and map the glyphs accordingly. If the available glyphs don't match the expected box-drawing characters, the coder should use ASCII fallbacks (`+`, `-`, `|`, `*`, `(`, `)`, etc.).
    
    **5c. Drawing primitives**:
    - `draw_box(uint8_t x, uint8_t y, uint8_t w, uint8_t h, chargfx_color_t color)`: Draw a filled rectangle box with border
    - `draw_line(char sx, char sy, char ex, char ey, chargfx_color_t color)`: Draw a line between two character-cell positions using Bresenham's algorithm
    - `draw_arrowhead(char x, char y, chargfx_color_t color, char direction)`: Draw an arrowhead at the end of a line
    
    **5d. Core `algo_draw()` function**:
    ```
    1. Fill panel background with CHARGFX_BG
    2. Draw panel border frame with CHARGFX_GRAY
    3. Draw "ALG ###" header at panel_x, panel_y in CHARGFX_YELLOW
    4. For each of the 6 operators:
       a. Get box position from DX7_OPS[]
       b. Draw operator box outline with CHARGFX_WHITE (or CHARGFX_GREEN if output op)
       c. Center operator number label inside the box
       d. If op_levels != NULL, draw level indicator below box
    5. For each connection in algo_connections[alg_idx]:
       a. Get source box center and dest box center
       b. Draw line between centers with CHARGFX_WHITE
       c. Draw arrowhead at dest end pointing toward dest box center
    6. Highlight output operator (Op1 for most algorithms, Op6 for Algorithm 16) with CHARGFX_GREEN border
    ```
    
    **5e. Output operator detection**:
    ```c
    // The output operator is the one with C==1 and D==1 in the algorithm
    // For most algorithms: Op1
    // For Algorithm 16 (series cascade): Op6
    static uint8_t get_output_operator(uint8_t alg_idx) {
        if (alg_idx == 15) return 6; // Algorithm 16: series cascade, output is Op6
        return 1; // All others: output is Op1
    }
    ```
    
    **5f. Line drawing strategy**:
    - Use Bresenham's line algorithm in character-cell coordinates
    - For horizontal segments: use `'-'` or `GLYPH_LINE_HORIZ`
    - For vertical segments: use `'|'` or `GLYPH_LINE_VERT`
    - For diagonal segments: use `'/'` (down-right) or `'\'` (down-left)
    - At junctions, pick the glyph that represents maximum connectivity
    - If chargfx font lacks proper box-drawing characters, fall back to ASCII alternatives

    ```c
    #include "algo_conn_table.h"

    void ui_widgets_draw_algo(uint8_t x, uint8_t y, uint8_t alg_idx, uint8_t op_msk)
    {
        /* Clear the panel area first (30 chars wide x 9 chars tall) */
        for (uint8_t ey = y; ey < y + 9u && ey < TEXT_HEIGHT; ey++) {
            for (uint8_t ex = x; ex < x + 30u && ex < TEXT_WIDTH; ex++) {
                chargfx_set_cursor(ex, ey);
                chargfx_set_foreground(CHARGFX_BG);
                chargfx_set_background(CHARGFX_BG);
                chargfx_putc(' ', false);
            }
        }
        /* Draw the algorithm diagram */
        algo_draw(x, y, 30u, 9u, alg_idx, NULL, op_msk);
    }
    ```

## 5. Risk Mitigation

1. **Glyph availability**: If `FONT_SPECIAL_CHARACTERS_BITMAP` doesn't have the expected box-drawing glyphs, the `algo_gfx.c` implementation will fall back to ASCII characters (`+`, `-`, `|`, `*`, `(`, `)`, `/`, `\`, `>`, `v`). The coder should examine the bitmap and adjust glyph mappings accordingly.

2. **Display layout**: The operator box positions are based on the DX7 front-panel layout. If they don't fit well on the ST7789 display, the positions can be adjusted in the `DX7_OPS[]` array.

3. **Algorithm connections**: The hardcoded connections are based on the canonical DX7 algorithm diagrams. If any connections are incorrect, they can be fixed in the `gen_algo.py` script and regenerated.
